# SoundHub

**SoundHub** — это одностраничное веб-приложение (SPA), посвящённое миру музыкального продакшена.  
Проект был создан как курсовая работа и объединяет в себе подборку информации о DAW, плагинах, музыкальном софте и полезных материалах для начинающих и продвинутых продюсеров.

---

## О проекте

SoundHub — это визуально оформленный информационный сайт, где собраны:
- краткие обзоры популярных DAW;
- карточки с музыкальными программами и плагинами;
- текстовые блоки с описаниями и пояснениями;
- адаптивные секции с визуальными акцентами;
- интерактивные элементы интерфейса.

Проект построен как SPA: весь контент подгружается и отображается на одной странице, а отдельные разделы формируются через модульную структуру JavaScript-компонентов.

---

## Стек

### Frontend
- **HTML5** — семантическая структура страницы.
- **CSS3** — стилизация интерфейса, адаптивная вёрстка, анимации, grid- и flex-раскладка.
- **JavaScript (ES Modules)** — построение интерфейса, рендер страниц, переиспользуемые компоненты, интерактивность.

### Сборка и разработка
- **Vite** — быстрый dev-сервер, сборка проекта и подготовка к деплою на статический хостинг.
- **GitHub Pages** — хостинг готовой статической версии проекта.
- **GitHub Actions** — автоматический деплой после push в репозиторий.

### Дополнительно
- **Google Fonts** — подключение шрифтов `Inter`, `Press Start 2P`, `Exo 2`, `JetBrains Mono`.
- **SVG/PNG favicon** — иконки сайта.
- **Адаптивная типографика и доступность** — настройки для разных экранов и системных предпочтений пользователя.

---

## Особенности

- **SPA-архитектура**: сайт работает как единое приложение с модульной генерацией контента.
- **Модульная структура**: компоненты вынесены в отдельные JS-файлы.
- **Адаптивный дизайн**: сайт корректно выглядит на десктопах, планшетах и телефонах.
- **Декоративная типографика**: контраст между пиксельным заголовочным шрифтом и читабельным текстовым шрифтом.
- **Интерактивные эффекты**: hover-анимации, переливающиеся разделители, визуальные акценты.
- **Попап для email**: локальный pop-up без отправки данных на сервер.
- **Доступность**: учтены `prefers-reduced-motion`, повышенный контраст и системные особенности отображения.

## Структура проекта

Примерно так организованы основные части проекта:

- `src/components/` — переиспользуемые UI-компоненты.
- `src/pages/` — сборка страниц.
- `src/data/` — текстовые данные и контент.
- `src/assets/` — изображения, иконки и шрифты.
- `src/styles/` — стили, переменные и глобальные настройки.
- `src/main.js` — точка входа приложения.

## Как запустить локально

1. Установить зависимости:
   ```bash
   npm install
   ```

2. Запустить dev-сервер:
   ```bash
   npm run dev
   ```

3. Открыть проект в браузере по адресу, который покажет Vite.

---

## Сборка проекта

Чтобы создать production-версию:

```bash
npm run build
```

После сборки проект будет готов к размещению на статическом хостинге.

---

## Деплой

Проект можно открыть через **GitHub Pages**.  
Для корректной работы при публикации в Pages в Vite используется настройка `base`, а сборка и публикация выполняются через GitHub Actions.
